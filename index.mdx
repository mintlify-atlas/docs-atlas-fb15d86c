Sumo Oracle is an R-based machine learning project that predicts the outcome of sumo matches using two complementary models: a Bayesian Generalized Linear Model (GLM) and a neural network. Both models are trained on historical wrestler attributes — weight, height, age, and win record — and can produce predictions for upcoming bouts.

The GLM model achieves approximately **70% prediction accuracy**, outperforming the neural network baseline on this dataset.

## How it works

<Steps>
  <Step title="Prepare your data">
    Collect wrestler attributes for both competitors in CSV format. The required columns are `weight1`, `weight2`, `wins1`, `wins2`, `age1`, `age2`, `height1`, `height2`, and `result`.
  </Step>
  <Step title="Load and preprocess">
    Use `sumo.read()` to load the CSV into a typed tibble. Historical rows with known results are used for training; rows where `result` is `NA` are treated as matches to predict.
  </Step>
  <Step title="Train both models">
    Fit the binomial GLM with `glm()` and train the neural network with `neuralnet()` on the historical data.
  </Step>
  <Step title="Generate predictions">
    Call `predict.glm()` and `predict()` on the undecided rows. Each model outputs a directional prediction — left wrestler wins or right wrestler wins.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Bayesian GLM" icon="scale-balanced">
    Binomial logistic regression via `MASS` — interpretable coefficients and ~70% accuracy.
  </Card>
  <Card title="Neural network" icon="network-wired">
    Configurable hidden layer topology via the `neuralnet` package with logistic activation.
  </Card>
  <Card title="CSV data ingestion" icon="file-csv">
    Simple flat-file data format — no database required. Add new matches by editing one file.
  </Card>
  <Card title="Model evaluation" icon="square-check">
    Built-in train/test splitting and confusion matrix reporting via `caret`.
  </Card>
  <Card title="R Markdown report" icon="file-lines">
    Reproducible analysis with `sumo.Rmd` — renders a full PDF report with plots.
  </Card>
  <Card title="Side-by-side comparison" icon="code-compare">
    Run both models on the same undecided matches and compare their outputs directly.
